1. Generación de números y variables aleatorias¶

Describir el algoritmo de Ziggurat para distribuciones con funcin de densidad decreciente y compararlo con otros mtodos para la generacin de valores de la normal.

El algoritmo de Ziggurat es un mtodo para generar valores aleatorios a partir de una funcin de densidad montona decreciente

  • Distribucin exponencia

  • Distribucin normal

Su nombre proviene de unos templos construidos en la antigua Mesopotamia.

Este algoritmo est basado el mtodo de rechazo:

  1. Generar puntos aleatorios dentro del rectngulo azul (distribucin uniforme)
  2. Descartar los puntos ($x_{n}$, $y_{n}$) donde $y_{n}$ > $f(x)$
  3. Tomar la coordenada x de cada uno de los puntos restantes

La eficiencia de este mtodo suele ser baja porque muchos puntos generados acaban siendo descartados.

Idea bsica del algoritmo de Ziggurat:

Minimizar el rea de rechazo con mltiples rectngulos

  • FDP decreciente $f(x)$, donde $x 0$

  • Dividir el rea debajo de la curva en $n-1$ rectngulos y una cola con la misma rea.

    $z_k (k=0,1,...,n-1)$, $z_0=0$.
    La cola como los puntos bajo la curva con ordenada inferior a $f(z_{n-1})$.

  • Elegir uniformemente una de las regiones y aplicamos el mtodo del rechazo en esa regin.

  • Ejemplo: $n=8$ (en prctica n puede alcanzar 64, 128 o 256)

  • Regin central de un rectngulo: la parte que se superpone al rectngulo superior en la direccin de la longitud. El rectngulo superior no tiene regin central.

Inicialización¶

  • Entrada: nmero de secciones n.
  • Calculamos por aproximacin $z_n$.
  • A partir de $z_n$ y el rea comn de las secciones, calculamos el resto de puntos $z_k$.
  • Calculamos $\sigma_k = z_{k-1} / z_k$, que es la fraccin del rectngulo k que est enteramente contenida en la curva. $\sigma_1 = 0$

Algoritmo¶

  • En los casos en que el nmero caiga fuera de la porcin central o en la cola de la distribucin se requieren clculos extra.
  • Estos casos son menos probables cuanto mayor sea n.
  • Para valores de n altos, estos casos son extremadamente raros (< 3% para n = 128), por lo que el algoritmo es muy eficiente.

Comparación con otros métodos¶

  • Mtodo de rechazo: Es ms simple pero puede ser muy ineficiente, ya que se pueden llegar a rechazar un gran nmero de puntos, y para cada uno de ellos hay que calcular $f(x)$, que suele ser costoso.

  • Mtodo de inversin: El clculo de la funcin inversa puede ser muy costoso, y la funcin a veces es no primaria.

  • Box-Muller: Es muy eficiente y ha sido utilizado durante mucho tiempo, pero requiere al menos el clculo de un logaritmo y una raz cuadrada para los valores generados.

def zigguratAlgorithmFunction():
    POINT_SIZE = 50
    FIGURE_SIZE = (20, 10)

    counter = {}
    for rand in random_numbers:
        round_rand_number = np.round(rand, 2) # round the number to 2 decimals
        counter[round_rand_number] = counter.get(round_rand_number, 0) + 1

    x_axis = sorted(list(counter.keys()))
    y_axis = list(map(lambda x : counter[x] / N * 100, x_axis))

    domain = np.linspace(-max(x_axis), max(x_axis), N)

    fig, ax = plt.subplots(figsize = (FIGURE_SIZE[0], FIGURE_SIZE[1]));

    ax.plot(x_axis, y_axis, color = 'b')

    ax.plot(domain, list( # domain is the x axis and the rest y axis
        map(
            lambda x: pdf_standard_normal_distribution(x), 
            domain
        )
    ), color = 'r', linewidth = 5)

    plt.show()
    return counter

def zigguratAlgorithmTable(counter):
    standard_normal_table = pd.DataFrame(
        data =[],
        index = np.round(np.arange(0, 1, .1), 2),
        columns = np.round(np.arange(0.00, .1, .01), 2)
    )

    for index in standard_normal_table.index:
        for column in standard_normal_table.columns:
            z = np.round(index+column, 2)
            value = 0
            for k in np.round(np.arange(min( random_numbers ), z, .01), 2):
                if counter.get( k ) is not None:
                    value = value + counter.get(k)
            standard_normal_table.loc[index, column] = value / N

    standard_normal_table.index = standard_normal_table.index.astype(str)
    standard_normal_table.columns = [str(column).ljust(4, '0') for column in standard_normal_table.columns]

    return standard_normal_table

Conclusión¶

  • El algoritmo de Ziggurat es una mejora del mtodo de rechazo.
  • Es uno de los algoritmos de generacin de distribuciones que mejor conserva el equilibrio entre eficiencia y aleatoriedad de los nmeros generados.
  • Requiere una inicializacin que puede ser costosa, pero slo se requiere hacer una vez por distribucin.
  • Slo funciona con distribuciones estrictamente decrecientes en $[0,+\infty)$ o simtricas.
